# backlog_rules.md

## Main Rule
All development is maintained in Backlog (in \docs\product\backlog.md) during every work related to feature, from idea to finished accept tested delivery, according to \docs\product\backlog_rules.md
Read this file before maintaining backlog. Details: Not all feature work triggers a change to the backlog, but feature number, feature name, feature status, and priority and planning changes will.

## Exception on previously done features
Features done in previous PI's are to be kept as-is in backlog and not comply to rules in this document - because they are already done and improving festure information provides no value for them.

## Backlog Definition
The backlog consists of these sections:
- List of PI's. Each containing the features included in PI.
  Ordered in PI order - PI1 first.
  Hence, list of PI and their features start with the Done PI's and features. This section will increase in size through time and PI's.
  List of PI's might end with future PI's not yet done, inclusing currently expected features. It's a roadmap, not an exact plan before priotized by Product Team, so features in a PI can change until PI has been started (or a feature fails inside a PI).
- Not yet prioritized features.
  List of features not yet refined or expected in a PI.
  Features in this section are ordered according to priority. If priority not yet exist then it's put at the end.

## Feature Definition
Each feature is defined with Feature Number, Feature Status and Feature Name (find it in features.md)
